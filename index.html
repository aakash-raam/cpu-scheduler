// ABOUT CPU SCHEDULING

CPU scheduling is the basis of multi-programmed operating systems. By switching the CPU among processes, the operating system can make the computer more productive. The objective of multiprogramming is to have some process running at all times to maximize CPU utilization.

  • Turnaround Time — Total time from submission to completion of a process
  • Waiting Time — Time a process spends waiting in the ready queue
  • Response Time — Time from submission to first response
  • Throughput — Number of processes completed per unit time
// ALGORITHM 01
First Come First Serve

// HOW IT WORKS

FCFS is the simplest CPU scheduling algorithm. Processes are dispatched according to their arrival time on the ready queue. It is a non-preemptive algorithm — once a process starts executing, it runs until completion.

  • Processes are served in order of arrival
  • Simple to understand and implement (FIFO queue)
  • Can suffer from the convoy effect — short processes stuck behind long ones
  • Average waiting time is often quite long

// ALGORITHM 02
Shortest Job First

// HOW IT WORKS

SJF selects the process with the minimum burst time next. In non-preemptive mode, the running process finishes before the next is selected. In preemptive mode (SRTF), if a new process arrives with a shorter remaining time, it preempts the current process.

  • Optimal for minimizing average waiting time
  • Non-preemptive: once started, runs to completion
  • Preemptive (SRTF): uses shortest remaining time
  • Can cause starvation for longer processes

// ALGORITHM 03
Priority Scheduling

// HOW IT WORKS

Each process is assigned a priority number. The CPU is allocated to the process with the highest priority (lowest number = highest priority in this simulator). In preemptive mode, a higher-priority process that arrives will preempt the currently running process.

  • Priority 1 = Highest priority (lower value = higher priority)
  • Non-preemptive: runs selected process to completion
  • Preemptive: new higher-priority process preempts current
  • Risk of indefinite blocking (starvation) for low-priority processes
  • Aging technique can prevent starvation

// ALGORITHM 04
Round Robin

// HOW IT WORKS

Round Robin is designed specifically for time-sharing systems. Each process gets a small unit of CPU time called a time quantum (or time slice). After each quantum, the process is preempted and added to the end of the ready queue.

  • Each process gets equal CPU time per cycle
  • Preemptive — ideal for time-sharing environments
  • Performance depends heavily on the quantum size
  • Large quantum → behaves like FCFS; small quantum → high context-switch overhead
  • No starvation — every process gets a turn
